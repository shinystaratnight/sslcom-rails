This document introduces generating a Key Pair and CSR, and answers questions you might have. Generating a Key Pair and CSR for a Compaq iTP Secure WebServer
You use the Compaq iTP Secure WebServer keyadmin utility to generate key pairs and add or delete certificates from the server key database.

This utility is located in the server bin.

To generate a public and private key pair and CSR for server:
  1. Launch keyadmin to generate the key pair. Enter the command on a single line. For example:
    $ bin/keyadmin -keydb keydb -mkpair -dn 'DN'(-length key_len)
    Where:
    • Keydb is the name of the key database that where the public and private key will be stored. If the database you specify is nonexistent, the server creates the database for you and displays an informational message, notifying you that the new database was created.
    • mkpair instructs the server to generate a random public key pair with a default length of 512 bits.
    • -dn 'DN' is the full distinguished name for the new key pair. Enclose this distinguished name in single quotes to protect it from being interpreted by the shell. Do NOT use any of the following characters in the distinguished name field: ! # $ % ^ & * + = < > ?
    • -length key_len specifies the length of the key in bits. This option allows you to control the size of the encryption key. The minimum key size is 512 bits. The maximum key size is 1024 bits, for domestic servers only.
    • When prompted, type random keystrokes. These random keystrokes are used to generate a unique public/private key pair.
    • For instructions to generate key pair for Atalla WebSafe, please refer to the iTP Secure WebServer manual.
  2. When prompted, enter a password to protect the key pair. This password should be at least eight characters long and contain at least one non-alphabetical character, such as a number or punctuation mark. Remember the security of your server is only as good as the security of the key-pair file and its password. You must enter this password when installing your Secure Server ID. After you enter the key database password, keyadmin creates the private and public parts of a new key pair and stores them in the key database.
  3. Run keyadmin -mkreq to generate a CSR:
    bin/keyadmin -mkreq filename -dn 'DN' (-life days) (-webmaster webmaster-name) (-phone webmaster-phone-number) (-software software)
    Where:
    • -mkreq filename generates a CSR and writes it to the specified file. A key pair must already reside in the database.
    • -life days specifies the length of time that the certificate will be valid. The default is 365 days. The span requested is inserted in the CSR. Note that the CA can adjust this life span when issuing the ID.
    • -webmaster,-phone,-software add the specified plain text fields to the CSR.
Keyadmin writes the public key and distinguished name to the specified CSR file. The information in this filename is encoded in PKCS #10 message format.
  1. Organization Name. Enter the name of your organization. This is the organization name you wrote down in Enrollment Step 1: Confirm Domain Name. The name you enter must be identical to the name listed as the owner of the domain name. VeriSign can only issue a Secure Server ID to the organization that owns the specified domain name. If you are unsure of the correct organization name to enter, go back to Enrollment Step 1: Confirm Domain Name to look up the domain name registration.
  2. Organizational Unit. If necessary, enter the name of your unit within the specified organization. This is usually the group using the Secure Server ID. For example, enter the name of your department or business unit.
  3. Common Name. Enter your web site's fully-qualified name. For example, www.company.com. This is known as your site's common name. Do not include the protocol, such as HTTP:// or FTP://.
  4. Country. Enter the two-letter country code for your country. You must use the correct ISO country code, other abbreviations will not be recognized. For example, the correct code for the United Kingdom is GB, not UK.
  5. State/Province. Enter the full name of your state or province. Do not abbreviate.
  6. Locality. Enter the name of your city, town, or other locality.
  7. Request File. Enter the name of the CSR file to create. This file will contain the CSR. (This is your public key file and does NOT contain your private key.) The CSR looks something like this:

  9. Go back to the enrollment pages and follow the instructions in Enrollment Step 4: Submit CSR to submit your CSR.
