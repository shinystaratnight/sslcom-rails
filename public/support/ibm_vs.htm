IBM Internet Connection Secure Server

This document introduces generating a Key Pair and CSR, and answers questions you might have. Generating a Key Pair and CSR for an IBM Internet Connection Secure Server
To generate a public and private key pair and CSR for an IBM Internet Connection Secure Server:

  1. Open the create key and request certificate form.
  2. Enter a name for the new key pair.
  3. Enter a name for the key ring. This is the name of the file where the key pair will be written.
  4. Enter the number of bits you want to use in the key.
  5. Select VeriSign as your CA (certificate authority).
  6. Enter the name of your organization. This is the organization name you wrote down in Enrollment Step 1:Confirm Domain. The name you enter must be identical to the name listed as the owner of the domain name. VeriSign can only issue a Secure Server ID to the organization that owns the specified domain name. If you are unsure of the correct organization name to enter, go back to Enrollment Step 1: Confirm Domain Name to look up the domain name registration.
  7. When prompted, save the CSR to a file. The default file name is certreq.txt. Note: Do not use the export keys and request certificate radio buttons when creating new keys. You should the CSR request manually.
  8. Enter a password to protect the key file. This password should be at least eight characters long and contain at least one nonalphabetical character, such as a number or punctuation mark. Remember, the security of your server is only as good as the security of the key-pair file and its password. You will need this password to install your Secure Server ID. If you lose your private key or forget the password, you will not be able to install the ID and will need to request and purchase a new ID from VeriSign. Note: If you do not select automatic logon, you must enter this password whenever you restart the server. Selecting automatic logon is recommended. Click Apply to generate the CSR.   9. When asked whether or not to automatically log in and send the CSR, decline.
  10. Back up your key file and CSR on a floppy disk and store the disk in a secure location. If you lose your private key or forget the password, you will not be able to install your Secure Server ID and will need to request and purchase a new one from VeriSign.
  11. Go back to the enrollment pages and follow the instructions in Enrollment Step 4: Submit CSR to submit your CSR.
Renewing Your Secure Server ID
You must renew your Secure Server ID annually. 30 days before your Secure Server ID's expiration date, VeriSign sends a notification to your technical and organizational contacts. Secure Server IDs can only be renewed during the 30 days prior to expiration. During this time, both your new and old certificates are active; allowing a smooth transition to the new certificate without any interruption in service. When you renew your Secure Server ID, the old default key in your keyring is replaced with a new key. The keyring does not change; only the default key does. To renew your Secure Server ID:
  1. Open the server configuration & administration forms.
  2. Select Security.
  3. Select Create Keys.
  4. Select Secure Server Certificate.
  5. Enter a name for the new key, such as 1997key . Do not change the key ring name: C:\WWW\BIN\keyfile.kyr should be your current key ring. If you change the name of the key ring, a new keyring is created along with the new key and VeriSign cannot accept that key for renewal.
  6. Enter the key ring password. (There is no key password, only the password for the existing key ring.)
  7. Enter the requested company information.
  8. Under Save certificate request to file, specify the file where you want to save the CSR, such as c:\1997request.txt.
  9. Click on Apply. This will generate a new CSR.
  10. Submit the new CSR to VeriSign and complete the renewal process.
  11. When you receive your new Secure Server ID, extract and install it according to your server's instructions.
Note: Keyrings from 4.1 can be used on 4.2; however, you will not be able to renew your VeriSign Secure Server ID.
