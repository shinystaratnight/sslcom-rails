CSR Generation
The Graphical PSE Tool enables creating PKCS#10 requests for certificates and storing them in a file. The requests can be used to order certificates, which then can be imported into PSE files. Moreover certificates can be exported from PSE files into separate files. The file exchange function e.g.can be used to request a certificate for the XsPRESSO Gateway server PSE.

Here you can generate a PKCS#10 request for a new certificate and save it in a file. After clicking Certificate - File Exchange - Request a dialogue box appears. Type in all requested information. Fill out the following fields: Subject
The name subject for the certificate that will be created.

Locality
e. g. the town

State or Province
Write it completely (e. g. Florida)

Organization Organizational Unit

Country
The two letter code for your Country in uppercase (e.g. DE for Germany).

Key length
You can select between 512, 1024 and 2048 bit key length for the key pair, that will be generated.

Private Key Password
The password for the certificate that will be created; this password will be required when the new certificate is used.If there is already a private key within the PSE file, specify the password of this key.Confirm Private Key Password
Write the password again. If there is already a private key within the PSE file, this field will be disabled.


Challenge Password
The challenge password may be required if you want to revoke your certificate.
Confirm Challenge Password

Save PKCS#10 Request as

Here you can specify the file name of the file where the PKCS#10. Request should be saved.

Finally press the Send button.
The PSE file remembers the pending request. You will see it in the PSE tree area.
You can save this PSE file including the pending request, if you can not obtain the certificate immediately.

Besides remembering the request in the PSE file, the PSE Tool creates a request file (in our example E:\home\psetool\myVeriSignTest.req). It is up to you, to get a certificate for this request and import it into the PSE file.

This is one possible way: Open the file with a common text editor and copy the contents into the VeriSign enrollment page.VeriSign will create a Digital Certificate. Copy this certificate into a text fileand use Certificate - File Exchange - Import to get the certificate into the PSE file.
