Stronghold Server

This document introduces generating a Key Pair and CSR, and answers questions you might have.

Generating a Key Pair and CSR for a Stronghold Server Stronghold keys and certificates (Secure Server IDs) are managed through three scripts: genkey, getca and genreq. These are part of the normal Stronghold distribution. Keys and certificates are stored in the directory$SSLTOP/private/, where SSLTOP is typically /usr/local/ssl. To generate a key pair and CSR for your server:

  1. Run genkey, specifying the name of the host or virtual host: genkey hostname. The genkey script displays the filenames and locations of the key file and CSR file it will generate:
    • key file: /usr/local/www/sslhostname.key
    • CSR file: /usr/local/www/sslhostname.cert
    Note: If you already have a key for your server, run genreq [servername] to generate only the CSR.
  2. Press Enter. The genkey script reminds you to be sure you are not overwriting an existing key pair and certificate.
  3. When prompted, enter a key size in bits. We recommend using the largest key size available: 1024 bits for domestic servers, 512 for foreign servers.
  4. When prompted, enter random key strokes. Stop when the counter reaches zero and genkey beeps. This random data to create a unique public and private key pair.
  5. When prompted, enter y to create the key pair and CSR.
  6. Select VeriSign as your CA.
  7. Enter the two-letter country code for your country. You must use the correct ISO country code, other abbreviations will not be recognized. For example, the correct code for the United Kingdom is GB, not UK.
  8. Enter the full name of your state or province. Do not abbreviate.
  9. Enter the name of your city, town, or other locality.
  10. Enter the name of your organization. This is the organization name you wrote down in Enrollment Step 1: Comfirm Domain Name. The name you enter must be identical to the name listed as the owner of the domain name. VeriSign can only issue a Secure Server ID to the organization that owns the specified domain name. If you are unsure of the correct organization name to enter, go back to Enrollment Step 1: Comfirm Domain Name to look up the domain name registration.
  11. Enter the name of your unit within the specified organization. This is usually the group using the Secure Server ID. For example, enter the name of your department or business unit.
  12. Enter your web site's fully-qualified name. For example, www.company.com. This is known as your site's common name. Do not include the protocol, such as HTTP:// or FTP://.
  13. When your finished entering the CSR data, genkey automatically creates the CSR which you must send to VeriSign during the enrollment.   14. Back up your key file and CSR on a floppy disk and store the disk in a secure location. If you lose your private key or forget the password, you will not be able to install your Secure Server ID and will need to request and purchase a new one from VeriSign.
  15. Go back to the enrollment pages and follow the instructions in Enrollment Step 4: Submit CSR to submit your CSR.
