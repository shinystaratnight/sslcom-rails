Generating a Key Pair and CSR Using Enterprise Server 4.x

This document introduces generating a Key Pair and CSR, and answers questions you might have.
  • Creating a Trust Database
  • Generate a Certificate Signing Request
  • Back up your key pair file
Creating a Trust Database using Enterprise Server

In the Server Administration page, click on "security" at the top, then click on "Create Database".

  1. Cryptographic Module: Specifies whether the certificate database is internal.
  2. Database Password: Specifies the certificate database password.
A certificate database is a key-pair and certificate database installed on the local host. When you use an internal token, the certificate database is the database into which you install the key and certificate. In Enterprise Server 4.0, each server instance (including the Enterprise Administration Server) has its own certificate/key pair which is referred to as a trust database.

A key-pair file contains both the public and private keys used for SSL encryption. You use the key-pair file when you request and install a certificate. The key-pair file is stored encrypted in the following directory:

server_root/alias/-key.db.

For more information on creating a database, you can click on the "help" option on the Create a Database page, or visit: http://home.netscape.com/eng/server/webserver/4.0/ag/esapuirf.htm#1096282


Generating a Certificate Signing Request using Enterprise Server

In the Server Administration page, click on "security", then click on "Request a Certificate".

After you create the Trust Database, you must create certificate request (CSR) and submit it online to VeriSign to obtain your certificate.
  1. New Certificate: Specifies that the certificate being requested is new.
  2. Certificate Renewal: Specifies that the certificate being requested is a renewal of an existing certificate. Choose the appropriate option.
  3. CA-email address: This can actually be your email address. Once you have generated the CSR, this file will be emailed to you.
  4. Cryptographic Module: Specifies the module to be used with the certificate.
  5. Key Pair File Password: Specifies the certificate database password.
  6. Requestor Name: Specifies the name under which the certificate will be issued.
  7. Telephone Number. Specifies the telephone number of the requestor.
  8. Common Name. Specifies the fully qualified domain used in DNS lookups (for example, www.netscape.com). This is the name in the URL that a browser uses to connect to your site. It is important that these two names are the same. This is very important. If you are not sure what name to use in this field, you can contact VeriSign technical support for assistance.
  9. Email Address. Specifies the business email address used for correspondence between you and the CA.
  10. Organization. Specifies the official, legal name of your company, educational institution, organization, and so on. VeriSign will verify this information with either a Dun & Bradstreet number, or an alternate proof or right document, such as your articles of incorporation or business license.
  11. Organizational Unit. Specifies a description of an organizational unit within your company. This field is optional.
  12. Locality. Specifies a description of the city, principality, or country for the organization. This field is optional.
  13. State or Province. Specifies the state or province in which the business is located.
  14. Country. Specifies the two-character abbreviation of your country name (in ISO format). The country code for the United States is US.
  15. OK. Saves your entries. You must restart the server in order for your changes to take effect.

The server generates a certificate request that contains your information. This is the file that you send to VeriSign during the enrollment online. For more information on requesting a certificate, you can click on the "help" option on the Request a Certificate page, or visit: http://home.netscape.com/eng/server/webserver/4.0/ag/esapuirf.htm#1096303

Back up your Key Pair File

It is imperative that you back up your key pair file. Please save this information on a floppy disk, or other removable media, and store it in a secure place, such as a safe or safe-deposit box.
