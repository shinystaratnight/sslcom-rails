This document introduces generating a Key Pair and CSR, and answers questions you might have.

Generating a Key Pair and CSR for a Connect Server

To generate a public and private key pair and CSR for a Connect Server:

  1. Open a terminal window to your server.
  2. Log in as root.
  3. Change directories to the server root directory.
  4. At the prompt, enter bin/cms/admin/bin/sec-key to run the key-generation program.
  5. When prompted, enter the path and file name where the new key-pair file should be written. Usually, the key-pair file is stored on the server root, in cms-<server_id>/config/ServerKey.db.
  6. When the key generation program displays a screen with a progress meter, type random keys at varying intervals until the progress meter is full. These random keystrokes are used to create unique public and private encryption keys for the server.
  7. When prompted, enter a password to protect the key pair. This password must be at least eight characters long and contain at least one nonalphabetical character, such as a number or punctuation mark. Remember, the security of your server is only as good as the security of the key-pair file and its password. Once you enable SSL, you must enter this password whenever you restart the server. You must also enter this password when installing your Secure Server ID.
  8. When prompted, confirm your password by re-entering it and clicking OK.
  9. From the Server Manager, select Encryption | Generate Server Key. The "Generate a Key Pair File" form displays.
  10. Enter the two-letter country code for your country. You must use the correct ISO country code, other abbreviations will not be recognized. For example, the correct code for the United Kingdom is GB, not UK.
  11. Enter the full name of your state or province. Do not abbreviate.
  12. Enter the name of your city, town, or other locality.
  13. Enter the name of your organization. This is the organization name you wrote down in Enrollment Step 1: Confirm Domain Name. The name you enter must be identical to the name listed as the owner of the domain name. VeriSign can only issue a Secure Server ID to the organization that owns the specified domain name. If you are unsure of the correct organization name to enter, go back to Enrollment Step 1: Confirm Doamin Name to look up the domain name registration.
  14. If applicable, enter the name of your unit within the specified organization. This is usually the group using the Secure Server ID. For example, enter the name of your department or business unit.
  15. Enter your web site's fully-qualified name. For example, www.company.com. This is known as your site's common name. Do not include the protocol, such as HTTP:// or FTP://.
  16. In the Key File Path field, enter the path and filename of the key-pair file. (The path is relative to the server root.) The key file should be in a protected directory that only the server has read and write access to.
  17. Click OK to generate the key-pair file and store it in the specified directory.
  18. Launch the web administration interface for your server.
  19. In the the encryption configuration section, select "Generate A Key".
  20. When prompted, enter the key-pair filename you specified when running the key generation program, sec-key.
  21. Select "Request A Certificate".
  22. When prompted, enter your email address in the "Certifying Authority" field.
  23. A certificate signing request will be generated using the key-pair and sent to your email address.

  24. Back up your key file and CSR on a floppy disk and store the disk in a secure location. If you lose your private key or forget the password, you will not be able to install your Secure Server ID and will need to request and purchase a new one from VeriSign.
  25. Go back to the enrollment pages and follow the instructions in Enrollment Step 4: Submit CSR to submit your CSR.
