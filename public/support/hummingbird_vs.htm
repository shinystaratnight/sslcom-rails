Obtaining a Site Certificate for SSL from VeriSign

Introduction
To obtain a site certificate from VeriSign, you must carefully follow all of the instructions at VeriSign's website. Allow plenty of time to complete this procedure and be sure all information you provide is accurate. During the enrollment process, you will be asked to provide a Certificate Signing Request (CSR). These instructions will help you generate the CSR for Vorlon, and then later install the certificate.
To prepare, first create the directories needed for SSL support in Vorlon:

  1. Create the following subdirectories in the vorlon install directory. Assuming Vorlon was installed in c:\vorlon, create these directories: 
    
	C:\vorlon\ssl\private
	C:\vorlon\ssl\certs
  2. Add c:\vorlon to the PATH, so that you will be able to execute the openssl program from any directory.
  3. Obtain or create the configuration file openssl.cnf in the ssl directory.
    NOTE: If this file is missing from your Vorlon distribution you should obtain it from Hummingbird technical support.
  4. Define an environment variable named HOME and set it to the ssl directory, c:\vorlon\ssl. On Windows NT, you would type: 
    set HOME=c:\vorlon\ssl

Choosing the Common Name for the Certificate
One of the important pieces of information required to obtain the site certificate is the common name you will use for your server. This should be the exact Fully Qualified Domain Name of the server (FQDN). For example, if users will type https://www.mycompany.com in their browser to reach your server, then your common name is www.mycompany.com. Wherever you see the text "common_name" in these instructions, substitute the actual name you have chosen.

Note: The certificate is tied to the common name you have chosen. If you choose the wrong common name, or if you use the resulting certificate on a different server, the browser will display a warning to the user. If the user accepts the warning, the certificate can still be used. Verisign will have additional help and instructions for choosing the common name.

Creating a private key
You will first need to generate a private key. You will need this key for several steps later.
  1. Start by finding a large, random file on your hard disk to use as a random seed enhancer. If possible, concatenate several such files together, and store the result in a file named randomfile in the c:\vorlon\ssl\private directory.
  2. Change to the c:\vorlon\ss\private directory and run the following command: 
    openssl genrsa -out common_name.key 
	-des3 -rand randomfile 1024
    Note: You will be prompted for a passphrase. Choose something that you will remember but that would be difficult for others to guess. Do not forget the passphrase!
  3. Your private key is now stored in the file common_name.key. You must protect this file. Make a backup copy to ensure that you will not lose it, and that other users cannot access it.

Creating the Certificate Signing Request (CSR)
You will next create a file containing all the information needed by VeriSign to generate a certificate for your site. This same file may also be used to generate a temporary self-signed certificate while you wait for VeriSign to process your enrollment request. (see instructions below).
  1. Change to the c:\vorlon\ssl\certs directory and run the following command: 
    openssl req -out common_name.csr 
-config ..\openssl.cnf -new -key 
..\private\common_name.key

Note: You will be prompted to answer several question about the site you want to create a certificate for. When asked for the common name, type in your chosen common name (i.e. what you are using for common_name).

Your Certificate Signing Request (CSR) is now stored in the file common_name.csr. You will use this file later. You should also make a backup copy of the CSR.

When prompted during the enrollment process, you will paste the entire contents of the CSR into your browser to send it to VeriSign. When VeriSign sends you your certificate, continue with the directions below for installing it in Vorlon.

Installing Your Certificate
When you receive your certificate in email, you must first extract the certificate to a text file name common_name.crt in the c:\vorlon\ssl\certs directory. Be sure to include the BEGIN CERTIFICATE and END CERTIFICATE lines. Now convert the certificate to the format used by Vorlon using the following commands:
  1. Change to the c:\vorlon\ssl\certs directory
  2. Run the following command: 
    
openssl pkcs12 -export -in common_name.crt -inkey
 ..\private\common_name.key -out file.p12 -name friendly_name

    where friendly_name is any friendly name that should be use for your certificate. The friendly name will be seen by users if they choose to install your certificate in their browser. If this command generates error messages, see the section "Certificates in Other Formats."
  3. Now generate the server.pem file used by Vorlon: 
    openssl pkcs12 -in file.p12 -nodes -out server.pem
  4. Save and protect your certificate file, common_name.crt!
  5. If you ever need to reintsall Vorlon, you can restore your private key and certificate and then follow these last instructions again to recreate the server.pem file.

NOTE: It is vitally important that your private key file, CSR, and certificate be backed up and protected from theft. You must also not forget your private key passphrase, or allow it to be stolen. Also protect the server.pem file from theft Failing to follow this advice could cost you additional money.

Creating a Self-Signed Certificate
If you want to create your own self-signed certificate, you may do so using the same CSR you submitted to VeriSign. You can use the self-signed certificate it as a temporary site certificate while you are waiting for VeriSign to process your enrollment.
  1. Change the current directory to c:\vorlon\ssl\certs and run the following command to generate a certificate from the CSR you created previously: 
    openssl req -out common_name.crt -x509 -key 
..\private\common_name.key -in common_name.csr -config ..\openssl.cnf

Your certificate is now stored in the file common_name.crt. Follow the certificate installation instructions for installing the certificate.

Configuring HTTPS Support in Vorlon
All that remains is to turn the HTTPS support in Vorlon. In all 1.x versions of Vorlon, you edit the server.ini and add the line: server.startHTTPS = true

In all version 2.x or above, edit the server.xml file in the config directory and add or edit the https tag within the server tag: 
<https enabled="true" port="443">

Certificates In Other Formats
It is possible that a mistake has been made and the CA has sent your certificate in an unexpected format. This may be the case if you had requested a Stronghold certificate from CIBC in Canada. If the pkcs12 command generates errors for your certificate, try running this command first: 
openssl pkcs7 -in common_name.crt -print_certs

This will print out the certificates. Cut and paste the one for your server and then run the pkcs12 command.

Users Requesting Certificates From an Intermediate CA
If your company resides outside the US, you may be required to process your VeriSign certificate enrollment through an intermediate certificate authority (CA) such as CIBC in Canada. In this case, some versions of Netscape will require that your obtain the intermediate certificate from the CA so that the browser can verify the certificate chain. Vorlon includes support for an intermediate certificate, simply by naming the intermediate certificate intermediate-server.pem and place in the same directory with the server.pem.
