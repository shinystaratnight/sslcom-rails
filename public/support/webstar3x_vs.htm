Generating a Key Pair and CSR for a 4D Inc. Webstar 3.x Server


Technical Note: Webstar Version 3.x and earlier used certificates that were create using the PEM format. The current certificates issued by Verisign use PKCS (the most common standard). To use the PKCS certificates, you will need to update to Webstar 4.x.


Use the 4D Inc. Key Generator application to generate a key file containing a public/private key pair for use with your secure server. Follow these steps:


  1. Open the SSL Tools folder and locate the Key Generator.
  2. Launch the Key Generator application by double clicking its icon.
  3. Enter a password to protect the use of your key pair. WebSTAR/SSL will ask for this password when you initially launch the server, so it can use this key pair.
  4. Click "Create Key Pair" to generate your public/private key pair. Generating a key can take a few minutes.
  5. The Public/Private Key pair is stored in an encrypted file named Public/Private Key.
  6. Move this file into the same folder as the WebSTAR/SSL server. It must be in the server folder to work.



Do not forget this password! You'll need it later to authorize WebSTAR/SSL to use your public/private key pair. We recommend that you write it down and store it in a safe place. If you lose the password, you will have to purchase a new Digital ID.

Generate a Digital ID Request

Use the Digital ID Request application (in the SSL Tools folder) to create a request for a Digital ID which you will send to VeriSign.

The application requires identifying information such as the name of your organization, department within the organization, and where the organization is located. It also requires the Public/Private Key for your site

Your Digital ID Request may be rejected if the information is not properly formatted. Be sure to closely follow the conventions outlined in the instructions. If any of the information is improperly formatted, VeriSign will ask you to correct it and send the request again.

Follow these steps:


  1. Locate the Digital ID Request application in the SSL Tools folder and launch it by double clicking its icon.
  2. Type the name of your Web site in the "Distinguished Name" field, for example:
    www.domain.com
    Make sure that the Distinguished Name you specify will be the actual host name of your WebSTAR/SSL server--it will be encoded into the Digital ID as part of your Distinguished Name and cannot be changed later without purchasing a new Digital ID.
    Furthermore, this host name should be the main "A name" entry for your machine on your DNS server. Your Digital ID may have problems if you use an IP address or if the host name is a "CNAME" entry (DNS alias), for example. Contact your network administrator for guidance if necessary.
  3. Type the name of your organization in the "Organization" field, for example: Widgets Corporation
  4. Type the name of the department or other organization unit in the "Org Unit" field, for example:Support
  5. Type the name of the city or town in which your organization is located in the "Locality" field, for example: Berkeley
  6. Type the name of the state or province in which the organization is located in the "State/Province" field, for example: California Do not abbreviate the state or province name: spell it out as shown above.
  7. Type a two-letter code for the country in which you are located in the "Country Code" field, for example: US Do not spell out the country name: use a 2-letter code. The code for the United States is US. For Canada, the code is CA.
  8. Type the email address of the site's Webmaster in the "Email Address" field, for example: webmaster@domain.com
  9. In the "Phone Number" field, type a phone number where VeriSign can reach you, and type your fax number in the "FAX Number" field.

    The Email Address, Phone Number, and FAX Number fields are not part of the Digital ID. VeriSign uses these fields to contact you if it finds a problem with the Digital ID request.
  10. Click "Select Key File" and select the Public/Private Key file you created. This must be the original Public/Private Key file. Once you have selected a Public/Private Key file, the key file and the Digital ID you will receive are a digital ID pair , and cannot be separated.

    If you lose the Public/Private Key file and generate a new one, your Digital ID will no longer match. You will have to send a request to VeriSign for a new Digital ID, which you may be charged for. For this reason, be sure to keep backup copies of your files.
  11. Enter the password required to access your public/private key pair (the password you entered when generating the key pair, as described in Generate a Public/Private Key ).
  12. Click the "Create" button to generate your encrypted Digital ID request form.


The application creates a file named Digital ID Request and then quits. Submit this file at the VeriSign Online Enrollment form.
