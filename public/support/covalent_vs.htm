Covalent Raven

Certificates and keys are managed by the Raven System Management Interface, found at /usr/local/raven/bin/ravenctl. Keys are stored by default at /usr/local/raven/module/pki/keys, and certs, by default, are stored at /usr/local/raven/module/pki/certs.

The following proceduredescribes the process required to generate a key pair and CSR for your Raven SSL server.
  1. Run ravenctl. Select 'Raven PKI Certificate Manager'
  2. Select Generate Certificate and Key. The following questions will be asked, press the Enter Key to accept the setting in the brackets.
Generating Your Key Pair

Name of the file to store certificate/key? [server.domain.com] -->
This is usually the fully qualified domain name of the domain you are securing.
Number of bits in key (choose 512 or 1024)? [1024] -->
At the prompt above, enter the number of bits that you want your key file to contain. More bits means that the key will be harder to crack but there will be more server overhead required to encrypt the data. Fewer bits means less
overhead for the server to encrypt the data, but makes the key easier to crack.
Generating 1024 bits of randomness -->
No input is needed. The key generation process provides an internal random
entropy generator. The process will create twice the number of random bits that
you have chosen for you key size.
Generating 1024 random bits based on measuring the time interval between your keystrokes.
Please enter random text on your keyboard. 1024 <- remaining. This creates yet another
random entropy pool. This process helps assure that your key is difficult to predict and
thereby crack. Enter random text untilthe value hits 0.
Enter PEM pass phrase:
Verifying password - Enter PEM pass phrase:
This pass phrase will be used to to encrypt your key as it is stored on disk. It is not
necessary to keep keys encrypted on disk and this adds to difficulty in automating
the startup process for the server since an encrypted key will require you to enter a
pass phrase during the server startup phase. The option to decrypt the key on disk
can be applied after the install through the Raven CTL manager.
Using configuration from /usr/local/raven/module/pki/lib/certtool.conf.
Enter PEM pass phrase:
Enter in the pass phrase you used to encrypt your key. Generating Your Certificate

Country Name (2 letter code) [US]:
State or Province Name (full name) [Some-State]:
Enter the full name of the state, not the abbreviation.
Locality Name (eg, city) [Some-City]:
Organization Name (eg, company) [Some-Company/Organization]:
This will need to be the same organization that owns the domain you are securing.
Organizational Unit Name (eg, section) [Secure Services Division]:
Common Name (eg, server name) [www.servername.com]:
Put the fully qualified domain name of the domain you are securing.
Email Address [webmaster@servername.com]:

Congratulations, the certificate and key pair have been successfully generated. Make sure to back up your key pair to disk and save it in another location. If the key pair is lost or corrupted your certificate will not work and you will need to request another Digital ID from Verisign.

After generating the certificate and key pair, you will need to create a Certificate Signing Request. The CSR is what you will cut and paste into the form at step 4 of the enrollment process.

Generating a Certificate Signing Request

From the 'Raven PKI Certificate Manager' screen select the 'Generate Certificate Signing Request' option. The following questions will be asked:
Generate certificate signing request (CSR) for? [domain.cert]:
Enter in the name of the certificate from the options given, that you would like to create the CSR from.
Is this CSR being sent to Verisign? -->
Select Y.
Enter PEM pass phrase:
Enter the phrase you used to encrypt the key.

The following questions will be used in your certificate. If you enter '.', the field will be left blank.
Country Name (2 letter code) [US]:
State or Province Name (full name) [Some-State]:
Make sure to use the full name of the state, NO abbreviations.
Locality Name (eg, city) [Some-City]:
Organization Name (eg, company) [Some-Company/Organization]:
This should be the same organization that the domain you are securing is registered to.
Organizational Unit Name (eg, section) [Secure Services Division]:
Common Name (eg, server name) [www.servername.com]:
Put the fully qualified domain name of the server/web site you are securing.
Email Address [webmaster@servername.com]:
Webmaster Email Address [webmaster@servername.com]:
Webmaster Phone Number [222-333-4444]:
Send CSR via Email to? [root@covalent.net]:
This is the email address where the CSR will be emailed.
