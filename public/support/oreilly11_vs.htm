This document introduces generating a Key Pair and CSR (Certificate Signing Request), and answers questions you might have.

Generating a Keypair and CSR for O'Reilly WebSite Professional 1.1
WARNING: BACK UP YOUR WEBSITE.KEY FILE. This is a crucial safety precaution in case anything goes wrong. The website.key file is located in C:\Website\Admin

  1. Run Certificate Manager (Certman) from the Start Menu, Programs, WebSite Professional.
  2. Enter the password and click OK to continue.
  3. Click the Request Certificate button.
  4. Enter the requested information. Make sure you enter the correct domain name. The wizard creates your CSR for this domain.
  5. When you've completed the wizard, close Certificate Manager.
  6. Back up your website.key file and CSR on a floppy disk and store the disk in a secure location. If you lose your private key or forget the password, you will not be able to install your Secure Server ID and will need to request and purchase a new one from VeriSign.
  7. Go back to the Verisign enrollment pages and follow the instructions in Enrollment Step 4 to submit your CSR.


Renewing Your Secure Server ID for WebSite Professional 1.1
You must renew your Secure Server ID annually. Thirty days before your Secure Server ID's expiration date, VeriSign sends a notification to your technical and organizational contacts. Secure Server IDs can only be renewed during the 30 days prior to expiration. During this time, both your new and old certificates are active, allowing a smooth transition to the new certificate without any interruption in service.

To renew your Secure Server ID:
  1. Back up your current certificate database ( C:\website\admin\website.key) to a floppy disk.
  2. Rename C:\website\admin\website.key to C:\website\admin\website.old.key.
  3. Open Certificate Manager and create a new CSR with the same Common Name as your original request.
  4. Back up the newly created website.key file to a different floppy disk.
  5. Rename the new C:\website\admin\website.key file to C:\website\admin\website.key.next. You will not be able to use this new key pair database until you receive a new Secure Server ID from Verisign.
  6. Rename C:\website\admin\website.old.key back to C:\website\admin\website.key
  7. Submit the new CSR to Verisign and wait for your new Server ID.
  8. When you receive your new Secure Server ID, rename C:\website\admin\website.key to C:\website\admin\website.old.key.
  9. Rename C:\website\admin\website.key.next to C:\website\admin\website.key.
  10. Import your new Secure Server ID, using the WebSite Certificate Manager.
